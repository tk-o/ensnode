Self-hosting gives you full control over how you index and serve ENS data. Compared to the public [Hosted Instances](/docs/hosted-instances), your own ENSNode instance lets you:

- **Select [ENS Namespace](/docs/hosted-instances#ens-namespaces)**: choose which ENS protocol deployment ENSNode will provide data for (ex: `mainnet` or `sepolia`). Great for testing and development.
- **Activate [ENSNode Plugins](/docs/hosted-instances#ensnode-plugins)**: adjust indexed data model and indexed data records for your specific use case.
- **Tune "name healing" coverage** — choose a [label set](/docs/services/ensrainbow/usage/available-label-sets#available-label-sets) that matches your requirements for [ENSRainbow](/docs/services/ensrainbow#how-ensrainbow-helps).
- **Customize your data model** — implement an [ENSDb Writer](/docs/integrate/integration-options/ensdb-writers) to control how data is indexed into your own [ENSDb instance](/docs/integrate/integration-options/ensdb).
- **Build your own APIs and pipelines** — implement an [ENSDb Reader](/docs/integrate/integration-options/ensdb-readers) to serve your apps directly from your ENSDb instance.
- **Control versioning, updates, and uptime** — decide when to adopt new releases and apply your own high-availability strategy.

## Quick start

The fastest way to run ENSNode is with Docker Compose:

```bash
# 1. Copy and edit the environment file
cp docker/envs/.env.docker.example docker/envs/.env.docker.local

# 2. Set your namespace, plugins, and RPC endpoints in the file above

# 3. Start the stack
docker compose -f docker/docker-compose.yml up -d
```

For local development against a devnet, you can skip configuration and use `docker/docker-compose.devnet.yml` instead.

:::tip[Skip the long backfill]
You can avoid reindexing the entire ENS history from scratch by bootstrapping from an [ENSDb snapshot](/docs/integrate/integration-options/ensdb-cli#ensdb-snapshots). Snapshots include an indexed state and RPC cache, so you only need to catch up on events after the snapshot was taken.
:::
